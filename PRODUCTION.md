# Production deploy guide

JobLens ATS deploys as:

| Layer | Host | Notes |
|-------|------|--------|
| Frontend | [Vercel](https://vercel.com) | Next.js app in `frontend/` |
| API | [Render](https://render.com) | FastAPI in `backend/` |
| Database | Render Postgres | Linked via `render.yaml` |

Repo: https://github.com/N-sfd/Joblens.git (`main` is current).

---

## Recommended order

1. Deploy **Render** (API + Postgres) and confirm `/health`.
2. Deploy **Vercel** with `BACKEND_URL` pointing at the Render service.
3. Set **Clerk** roles and production keys.
4. Register **Zoho** production redirect URI and reconnect Mail.
5. Optionally **migrate** local SQLite/Postgres data into Render Postgres.
6. Run the **smoke checklist**.

Do not skip CORS (`ALLOWED_ORIGINS`) or Clerk (`ATS_AUTH_ENFORCE=true`) — ATS will fail open or closed incorrectly.

---

## 1. Backend on Render

### Blueprint (preferred)

1. Render Dashboard → **New** → **Blueprint**.
2. Connect `N-sfd/Joblens` and apply `render.yaml`.
3. Approve:
   - `joblens-db` (Postgres)
   - `joblens-api` (web service)
4. Fill every secret marked `sync: false` (table below).
5. Deploy. Start command already runs:

```text
python -m alembic upgrade head && uvicorn main:app --host 0.0.0.0 --port $PORT
```

6. Open `https://<joblens-api>.onrender.com/health`  
   Expect: `{"status":"healthy"}`

### Manual service (alternative)

- **Root Directory:** `backend`
- **Build:** `pip install -r requirements.txt`
- **Start:** same as above
- Create a Render Postgres instance and set `DATABASE_URL` from its **Internal** connection string (or External if migrating from your laptop).

### Render environment variables

| Variable | Required | Example / notes |
|----------|----------|-----------------|
| `DATABASE_URL` | Yes | Auto from blueprint (`joblens-db`) |
| `GROQ_API_KEY` | Yes | Resume/job AI parsing |
| `ALLOWED_ORIGINS` | Yes | `https://your-app.vercel.app` (comma-separate if multiple) |
| `CLERK_SECRET_KEY` | Yes | `sk_live_…` (or `sk_test_…` while validating) |
| `CLERK_JWKS_URL` | Yes | `https://<instance>.clerk.accounts.dev/.well-known/jwks.json` |
| `CLERK_ISSUER` | Yes | `https://<instance>.clerk.accounts.dev` |
| `ATS_AUTH_ENFORCE` | Yes | `true` (set by blueprint) |
| `ENV` | Yes | `production` (set by blueprint) |
| `SECRET_KEY` | Yes | Auto-generated by blueprint |
| `ZOHO_CLIENT_ID` | For Mail | Zoho API console |
| `ZOHO_CLIENT_SECRET` | For Mail | Zoho API console |
| `ZOHO_REDIRECT_URI` | For Mail | `https://your-app.vercel.app/ats/settings/zoho/callback` |
| `ZOHO_ACCOUNTS_BASE_URL` | Optional | Default `https://accounts.zoho.com` (use `.eu` / `.in` if needed) |
| `ZOHO_MAIL_API_BASE_URL` | Optional | Default `https://mail.zoho.com` |
| `TOKEN_ENCRYPTION_KEY` | For Mail | Fernet key — **reuse local key** if you migrate encrypted Zoho tokens |
| `ANTHROPIC_API_KEY` | Optional | Only if you switch off Groq |
| `STORAGE_PROVIDER` | Optional | `local` (ephemeral on Render) |
| `AI_RATE_LIMIT_PER_MINUTE` | Optional | Default `20` |
| `ZOHO_RATE_LIMIT_PER_MINUTE` | Optional | Default `30` |

Generate a Fernet key:

```powershell
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

## 2. Frontend on Vercel

1. Vercel → **Add New Project** → import `N-sfd/Joblens`.
2. **Root Directory:** `frontend`.
3. Set environment variables (Production + Preview if you use preview deploys):

| Variable | Required | Value |
|----------|----------|--------|
| `BACKEND_URL` | Yes (recommended) | `https://<joblens-api>.onrender.com` — **no** trailing slash, **no** `/api` |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Yes | `pk_live_…` / `pk_test_…` |
| `CLERK_SECRET_KEY` | Yes | Same Clerk secret as backend |
| `NEXT_PUBLIC_CLERK_SIGN_IN_URL` | Yes | `/sign-in` |
| `NEXT_PUBLIC_CLERK_SIGN_UP_URL` | Yes | `/sign-up` |
| `NEXT_PUBLIC_API_URL` | Optional | Only if you call the API cross-origin instead of the proxy |

4. Deploy.
5. If visitors hit a Vercel login wall: **Settings → Deployment Protection** → disable for Production (or allow your domain).

After the first Vercel URL is known, update Render `ALLOWED_ORIGINS` and Zoho `ZOHO_REDIRECT_URI`, then redeploy the API if needed.

## 3. Clerk roles (required for ATS writes)

With `ATS_AUTH_ENFORCE=true`, every ATS API call needs a valid Clerk JWT. Roles come from **`public_metadata.role`**:

| Role | Can |
|------|-----|
| `viewer` | Read only |
| `recruiter` | Create / update (not delete, not Zoho disconnect) |
| `admin` | Full access including deletes |

Set your user to admin (from `backend/` with `CLERK_SECRET_KEY` in `.env`):

```powershell
cd backend
python scripts\set_clerk_role.py admin --email you@company.com
```

Or in Clerk Dashboard → Users → user → **Public metadata**:

```json
{ "role": "admin" }
```

Sign out and back in after changing roles so a fresh session token is issued.

## 4. Zoho Mail (production)

1. Zoho API Console → your client → **Authorized Redirect URIs** → add:

   `https://<your-vercel-app>.vercel.app/ats/settings/zoho/callback`

2. Match that exact URL in Render `ZOHO_REDIRECT_URI`.
3. Use the correct regional hosts if your org is not US (`.eu`, `.in`, etc.).
4. In the app: `/ats/settings/zoho` → Connect → Sync.

If you migrate `zoho_connections` from local DB, `TOKEN_ENCRYPTION_KEY` on Render **must** match the key used when tokens were encrypted. Otherwise reconnect Zoho fresh.

## 5. Data migration

### Local Docker Postgres (already used in development)

```powershell
# Start DB (host port 5433)
docker compose -f docker-compose.postgres.yml up -d

cd backend
# In .env:
# DATABASE_URL=postgresql://joblens:joblens@127.0.0.1:5433/joblens
python -m alembic upgrade head
python scripts\migrate_sqlite_to_postgres.py --truncate
```

Source file: `backend/aijob.db` (gitignored).

### Copy local data into Render Postgres

1. Render → `joblens-db` → copy **External Database URL**.
2. From your machine (PowerShell):

```powershell
cd backend
$env:DATABASE_URL = "postgresql://...external-url-from-render..."
python -m alembic upgrade head
python scripts\migrate_sqlite_to_postgres.py --sqlite .\aijob.db --truncate
```

`--truncate` wipes target ATS tables first — only use on a fresh or disposable prod DB.

3. Unset the override when done: `Remove-Item Env:DATABASE_URL` (or close the shell).
4. Keep using the Render **Internal** URL for the running service (blueprint already sets this).

## 6. Smoke checklist

- [ ] `GET /health` on Render returns healthy
- [ ] Public site loads (`/`, resume, match) without Clerk
- [ ] `/ats` redirects to sign-in; after login, dashboard loads
- [ ] Your user can create/edit (role `admin` or `recruiter`)
- [ ] Zoho connect + sync imports mail
- [ ] Classify email → create job → matches → Send Job → submission
- [ ] Interview / offer / onboarding pages update status
- [ ] CORS: browser console has no blocked origin errors

---

## 7. Known production limits

| Topic | Reality |
|-------|---------|
| Resume files | `STORAGE_PROVIDER=local` uses the container disk — **lost on redeploy**. Move to Supabase/S3 before heavy use. |
| Free Render | Cold starts; first request after idle can take ~30–60s. |
| Free Postgres | Limited storage; take backups before large imports. |
| Secrets | Never commit `.env`, `*.db`, or live API keys. |

## 8. Troubleshooting

| Symptom | Likely fix |
|---------|------------|
| ATS API `401 Authentication required` | Sign in; confirm Clerk keys match frontend/backend; `ATS_AUTH_ENFORCE=true` |
| ATS writes `403` | Set `public_metadata.role` to `admin` or `recruiter` |
| Browser `Failed to fetch` / CORS | Set `ALLOWED_ORIGINS` to the exact Vercel origin (`https://…`) |
| UI calls localhost in production | Set Vercel `BACKEND_URL` (not `NEXT_PUBLIC_API_URL` to localhost) |
| `/api/api/...` 404 | `BACKEND_URL` must **not** end with `/api` |
| Zoho authorize `503` | Missing `ZOHO_CLIENT_ID` / `SECRET` / `REDIRECT_URI` on Render |
| Zoho reconnect fails after migrate | Mismatched `TOKEN_ENCRYPTION_KEY` — reconnect or remigrate with same key |
| Empty ATS after deploy | Schema migrated but data not copied — run section 5 against Render External URL |
| Alembic errors on boot | Check Render logs; ensure `DATABASE_URL` is Postgres (`postgres://` is auto-normalized) |

## Quick reference — local vs production

| | Local | Production |
|--|-------|------------|
| Frontend | `npm run dev` → `:3000` | Vercel |
| API | `uvicorn` → `:8000` | Render |
| DB | Docker Postgres `:5433` or SQLite | Render Postgres |
| Auth | Clerk test keys; `ATS_AUTH_ENFORCE=true` recommended | Clerk + enforce **on** |
| Zoho redirect | `http://localhost:3000/ats/settings/zoho/callback` | `https://<vercel>/ats/settings/zoho/callback` |
